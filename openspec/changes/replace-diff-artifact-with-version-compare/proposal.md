# Change: Replace Diff Artifact With Version Compare

## Why

Agent-created `diff` artifacts are weak product behavior. They ask the LLM to manufacture a patch-shaped artifact, but Agent-Conference already has append-only artifact versions and file-write approval diffs that are more deterministic and easier for users to trust.

The platform should stop presenting `diff` as a normal artifact output target. Diffs should be generated by Agent-Conference from two concrete artifact versions, not authored by an agent as a standalone delivery artifact.

## What Changes

- Remove `diff` from the `write_artifact` tool schema and LLM-facing descriptions.
- Keep existing `diff` artifact content parsing and rendering as read-only legacy compatibility.
- Add an artifact version comparison mode in the preview panel.
- Generate text diffs from stored artifact versions for supported content types.
- Show unsupported compare cases explicitly instead of asking an agent to invent a patch.
- Update OpenSpec and numbered specs so `diff` is legacy-only and version compare is the primary artifact diff experience.

## Out Of Scope

- Removing `diff` from `ArtifactType` or migrating old database rows.
- Applying legacy `diff` artifacts to target artifacts.
- Pending edit proposal / approval flows for artifact changes.
- New persistence tables for artifact compare results.
